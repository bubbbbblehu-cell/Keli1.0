## 📱 项目概述

### 产品定位
一款专为女性用户设计的安全地图应用，通过热力图展示用户评价的酒店安全信息，并集成AI拍照检测功能，帮助用户识别酒店环境中的安全隐患。

### 目标用户
- 独自出行的女性旅客
- 关注住宿安全的用户群体
- 需要快速评估酒店安全性的商务人士

### 核心价值
1. **可视化安全信息**: 通过热力图直观展示区域安全评分
2. **AI智能检测**: 使用Coze AI分析酒店房间照片，识别安全隐患
3. **用户评价系统**: 众包模式收集真实用户反馈
4. **多城市支持**: 覆盖国内外热门旅游城市

## 🎯 核心功能模块

### 1. 城市选择模块
**功能描述**: 应用启动时展示城市选择页面，用户选择目标城市后进入地图页面。

**支持城市**:
- 🇨🇳 中国: 西双版纳、贵阳、上海
- 🇹🇭 泰国: 曼谷
- 🇮🇹 意大利: 那不勒斯

**交互流程**:
1. 首次启动 → 显示城市选择页面
2. 选择城市 → 保存到 AsyncStorage/LocalStorage
3. 跳转到地图页面 → 显示该城市的酒店数据

### 2. 地图展示模块
**功能描述**: 使用热力图和标记点展示酒店安全评分分布。

**核心功能**:
- ✅ 热力图显示 (可开关)
- ✅ 酒店标记点 (颜色编码: 绿/黄/红)
- ✅ 点击标记查看酒店详情
- ✅ 评分筛选 (最低评分过滤)
- ✅ 城市切换
- ✅ 统计信息展示 (酒店总数、已评分数)

**数据来源**:
- 优先从 Supabase 数据库加载
- 失败时回退到本地模拟数据

**热力图算法**:
```javascript
// 根据安全评分生成热力图颜色
score >= 4.5 → 绿色 (#4ade80)
score >= 4.0 → 黄色 (#fbbf24)
score < 4.0  → 红色 (#f87171)
```

### 3. 酒店详情模块
**功能描述**: 展示酒店详细信息，支持收藏和评价。

**信息展示**:
- 酒店名称
- 安全评分 (0-5.0)
- 评价数量
- 地理位置

**用户操作**:
- ⭐ 收藏/取消收藏
- 📝 添加评价 (1-5星 + 文字评论)
- 🔗 跳转到在线平台 (Booking.com)

### 4. 拍照检测模块
**功能描述**: 使用AI技术检测酒店房间的安全隐患。

**检测部位** (6个):
1. **门锁**: 检测松动、撬动痕迹
2. **窗户**: 检测外部攀爬风险
3. **镜子**: 检测双面镜偷拍
4. **浴室**: 检测通风口隐藏摄像头
5. **插座**: 检测针孔摄像头
6. **路由器**: 检测网络嗅探设备

**工作流程**:
```
1. 选择检测部位 (多选)
   ↓
2. 安全自查引导 (问答形式)
   ↓
3. 拍摄照片上传 (每个部位1张)
   ↓
4. 提交到后端API
   ↓
5. Coze AI分析 (或模拟分析)
   ↓
6. 返回分析结果
   - 风险等级: high/medium/low
   - 安全评分: 0-5.0
   - 检测项: 具体问题列表
   - 建议: 安全建议
```

**AI分析结果格式**:
```json
{
  "hasRisk": true,
  "riskLevel": "high",
  "confidence": 0.85,
  "detectedItems": ["松动", "摇晃"],
  "recommendations": ["建议立即联系酒店前台"],
  "safetyScore": 2.0
}
```

### 5. 个人中心模块
**功能描述**: 管理用户收藏和个人信息。

**功能列表**:
- 📋 我的收藏 (收藏的酒店列表)
- 👤 个人资料 (头像、昵称、邮箱)
- 📝 评价记录 (待实现)

## 🗄️ 数据库设计

### 表结构概览

#### 1. cities (城市表)
```sql
- id: UUID (主键)
- code: VARCHAR(50) (城市代码, 如 'xishuangbanna')
- name: VARCHAR(100) (城市名称, 如 '西双版纳')
- country: VARCHAR(100) (国家)
- latitude/longitude: DECIMAL (坐标)
- is_active: BOOLEAN (是否启用)
```

#### 2. hotels (酒店表)
```sql
- id: UUID (主键)
- city_id: UUID (外键 → cities)
- name: VARCHAR(200) (酒店名称)
- latitude/longitude: DECIMAL (坐标)
- safety_score: DECIMAL(3,2) (安全评分 0-5)
- review_count: INTEGER (评价数量)
- is_active: BOOLEAN (是否启用)
```

#### 3. reviews (评价表)
```sql
- id: UUID (主键)
- hotel_id: UUID (外键 → hotels)
- user_id: UUID (外键 → auth.users)
- rating: DECIMAL(2,1) (评分 1-5)
- comment: TEXT (评价内容)
- photos: TEXT[] (照片URL数组)
- is_visible: BOOLEAN (是否可见)
```

#### 4. favorites (收藏表)
```sql
- id: UUID (主键)
- user_id: UUID (外键 → auth.users)
- hotel_id: UUID (外键 → hotels)
- UNIQUE(user_id, hotel_id)
```

#### 5. photo_detections (拍照检测记录表)
```sql
- id: UUID (主键)
- user_id: UUID (外键 → auth.users)
- hotel_id: UUID (外键 → hotels, 可选)
- photo_url: TEXT (照片URL)
- detection_result: JSONB (AI检测结果)
- safety_score: DECIMAL(3,2) (检测评分)
- status: VARCHAR(20) (pending/processing/completed/failed)
```

### 触发器
- ✅ 自动更新 `updated_at` 字段
- ✅ 评价后自动更新酒店评分和评价数
- ✅ 点赞后自动更新评价有用数

### 视图
- `hotel_details`: 酒店详情视图 (包含统计信息)
- `user_favorites_view`: 用户收藏列表视图

## ⚠️ 存在的问题

### 1. 架构问题

#### 1.1 代码重复严重
- **问题**: `index.html` 包含完整应用逻辑 (3000+ 行)，与 React Native 代码功能重复
- **影响**: 维护成本高，修改需要同步两处
- **建议**: 统一技术栈或明确分离 Web/Mobile 版本

#### 1.2 状态管理混乱
- **问题**: 
  - React Native 使用 AsyncStorage
  - Web 版使用 LocalStorage
  - 没有统一的状态管理方案
- **影响**: 数据同步困难，逻辑分散
- **建议**: 引入 Redux/Zustand 统一状态管理

#### 1.3 数据流不清晰
- **问题**: 
  - 数据库加载逻辑分散在多个文件
  - 模拟数据和真实数据混用
  - 错误处理不统一
- **影响**: 难以追踪数据来源，调试困难
- **建议**: 建立清晰的数据层抽象

### 2. 功能问题

#### 2.1 地图加载问题
- **问题**: 
  - 地图初始化时机不确定
  - 热力图和标记点有时不显示
  - 城市切换后地图未正确更新
- **原因**: 
  - DOM 渲染时机问题
  - 异步数据加载未等待
  - Leaflet 实例未正确清理
- **修复**: 已在 `index.html` 中添加大量日志和延迟处理

#### 2.2 拍照检测功能不完整
- **问题**:
  - 只有流程框架，缺少结果展示页面
  - AI 分析结果未持久化
  - 无法查看历史检测记录
- **建议**: 
  - 添加结果详情页面
  - 保存到数据库 (photo_detections 表)
  - 在个人中心显示历史记录

#### 2.3 用户认证缺失
- **问题**: 
  - 没有登录/注册功能
  - 使用匿名用户 ID
  - 收藏和评价无法跨设备同步
- **影响**: 用户体验差，数据无法持久化
- **建议**: 集成 Supabase Auth

### 3. 性能问题

#### 3.1 图片上传未优化
- **问题**:
  - 前端未压缩图片
  - 后端处理慢 (Sharp 同步处理)
  - 文件存储在本地 (不可扩展)
- **建议**:
  - 前端压缩后上传
  - 后端异步处理
  - 迁移到云存储 (Supabase Storage)

#### 3.2 数据库查询未优化
- **问题**:
  - 每次加载所有酒店数据
  - 未使用分页
  - 未使用 PostGIS 空间查询
- **建议**:
  - 按视野范围查询
  - 实现分页加载
  - 使用空间索引优化查询

### 4. 代码质量问题

#### 4.1 文档过多且重复
- **问题**: 28个 .md 文件，内容重复，难以维护
- **建议**: 合并为 5-6 个核心文档

#### 4.2 错误处理不完善
- **问题**:
  - 大量 `console.log` 用于调试
  - 错误信息不友好
  - 缺少统一的错误处理机制
- **建议**:
  - 使用日志库 (winston)
  - 统一错误处理中间件
  - 用户友好的错误提示

#### 4.3 测试缺失
- **问题**: 没有任何单元测试或集成测试
- **影响**: 重构风险高，回归问题多
- **建议**: 添加 Jest 测试
